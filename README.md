# WUT EOPSY Lab 3 - Synchronization

This repository contains the solution for Lab 3 of the EOPSY course. The primary goal of this assignment is to implement process synchronization mechanisms in a multiprocess Linux environment using POSIX shared memory and semaphores.

## Project Description

The program models a producer-consumer problem with a shared 16-element FIFO queue:
- Three independent producers generate elements of types A, B, and C.
- Two consumers process the elements according to specific extraction rules based on the current queue contents.

To avoid active waiting (busy waiting), the implementation utilizes a targeted dispatch approach. A centralized dispatch logic evaluates the queue state after every modification and awakens only the specific consumer that is eligible to process the current data using personal semaphores.

## Execution Modes

The executable accepts several command-line arguments to demonstrate different synchronization scenarios, including race conditions and proper concurrent behavior:

- `--no-sync`
  Disables semaphore operations. Used to demonstrate race conditions and data corruption when multiple processes access shared memory without proper synchronization.

- `--delay`
  Injects artificial delays between producer operations. This makes race conditions more apparent when synchronization is disabled.

- `--verbose`
  Enables detailed logging of queue state changes and process actions.

- `[count_a] [count_b] [count_c]`
  Three optional integer arguments to specify the exact number of elements to be generated by producers A, B, and C respectively.
